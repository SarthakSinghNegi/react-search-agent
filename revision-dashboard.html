Why Pydantic?

So output is clean and consistent. Instead of one big text blob, you get a clear object with known fields.

What you defined

Source has `url`. AgentResponse has `answer` and `sources: List[Source]`.

How it is enabled

Pass `response_format=AgentResponse` in `create_agent(...)`.

Where to read it

Use `result["structured_response"]` for real typed output.

Before`print(result["messages"][-1].content)` prints raw JSON-like text.
AfterUse `structured_response.answer` and loop through `structured_response.sources`.
ResultReadable output for terminal, API, and frontend use.
Invalid OpenAI key (401)

App cannot call model. Fix by updating `OPENAI_API_KEY` in `.env`.

LangSmith forbidden (403)

Tracing key/permissions issue. Agent still runs. Either fix key or set tracing false.

Module not found

Package missing in environment. Run `uv add ...` and `uv sync`.

Unexpected indent / syntax issues

Usually from pasted duplicate blocks or wrong indentation. Keep one clean invoke dictionary.

LangSmith: Why Output Looks Different by Node

Top-level node and ChatOpenAI node show different things. This is expected behavior.

Top-level LangGraph node

Shows full workflow result. This is where `structured_response` appears with separate `answer` and `sources`.

ChatOpenAI node

Shows only one model call output + API metadata like tokens and model name. Not the final shaped object.

Where did answer/sources go?

Under top-level output, often inside Additional Fields depending on LangSmith UI version.

Common Error Decoder

Fast lookup table: what it means and what to do.

"python was not found in package registry"

You ran `uv add ... python ...`. `python` is not added like normal package here. Use `python-dotenv`, not `python dotenv`.

Deleted `.venv` and app broke

Normal. `.venv` is re-creatable. Run `uv sync` or `uv run python main.py`.

Output became ugly after response_format

You printed raw message content. Print `result["structured_response"]` fields instead.
